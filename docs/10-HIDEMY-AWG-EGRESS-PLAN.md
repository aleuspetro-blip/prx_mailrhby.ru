# Hide.mn AmneziaWG: план исходящего VPN-туннеля

Дата: 20.07.2026

## Исходные данные

Новый VPS имеет публичный IPv4 `185.237.186.95`, Ubuntu 24.04, root-доступ, свободные TCP/UDP 443 и достаточные ресурсы.

Контрольные HTTPS-сайты доступны, но TCP-подключения ко всем проверенным адресам Telegram завершаются тайм-аутом. ICMP до части адресов Telegram проходит. Это указывает на фильтрацию TCP-доступа к Telegram, а не на общую потерю интернет-связности.

## Цель VPN

VPN нужен не как публичная точка входа для пользователей. Точка входа будущего MTProto Proxy остаётся на публичном IP VPS:

```
Telegram client -> 185.237.186.95:443 -> MTProto Proxy -> AmneziaWG -> Telegram DC
```

## Критическое правило

Нельзя включать конфигурацию Hide.mn как полный туннель `0.0.0.0/0` без адаптации. Полный туннель способен изменить маршрут ответов SSH и будущего MTProto Proxy, вызвать асимметричную маршрутизацию и потерю доступа к VPS.

Используется split tunneling: через AmneziaWG направляются только сети Telegram и служебные адреса, необходимые прокси. SSH, DNS, GitHub, обновления ОС и ответы клиентам MTProxy продолжают идти через `eth0` и публичный IP VPS.

## Выбранная реализация

Hide.mn официально предлагает для Linux конфигурацию AmneziaWG с обфускацией и указывает путь `/etc/amnezia/amneziawg/wg0.conf`, а подключение выполняется службой `awg-quick@wg0.service`.

Установка клиента выполняется из официального PPA Amnezia:

1. Установить зависимости и headers текущего ядра.
2. Добавить `ppa:amnezia/ppa`.
3. Установить пакет `amneziawg`.
4. Проверить DKMS-модуль и утилиты `awg`, `awg-quick`.
5. Не запускать VPN до проверки и адаптации полученного конфигурационного файла.

## Работа с секретами

Конфигурация Hide.mn содержит приватный ключ и не должна сохраняться в GitHub, логах проекта или публичных сообщениях. В документацию заносятся только схема маршрутизации, команды установки и результаты тестирования без ключей.

## Следующий этап

После установки AmneziaWG пользователь генерирует в Hide.mn конфигурацию `With obfuscation (AmneziaWG 2.0 client)` и загружает её на VPS через SFTP. Перед запуском:

- приватные ключи скрываются из диагностического вывода;
- проверяется Endpoint;
- `AllowedIPs` заменяется на split-туннель для Telegram;
- сервис сначала запускается вручную без автозапуска;
- проверяется сохранение SSH;
- проверяется маршрут к Telegram через `awg0`;
- только после успешных тестов включается автозапуск.
