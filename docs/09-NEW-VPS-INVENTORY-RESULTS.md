# Результаты первичной проверки нового VPS

Дата проверки: 20.07.2026

## Назначение

Новый VPS рассматривается как выделенный сервер для личного Telegram MTProto Proxy. На этом этапе изменения конфигурации, установка пакетов и запуск прокси не выполнялись.

## Подтверждённая конфигурация

- ОС: Ubuntu 24.04.2 LTS (Noble Numbat).
- Ядро: Linux 6.8.0-63-generic.
- Виртуализация: Microsoft Hyper-V, полная виртуализация.
- CPU: 2 vCPU, Intel Xeon E5-2658 v4 @ 2.30 GHz.
- RAM: 1.9 GiB, доступно около 1.5 GiB.
- Swap: `/swapfile` 2 GiB и zram около 951 MiB.
- Диск: 40 GB, корневой ext4-раздел около 39 GB; свободно около 33 GB.
- Публичный IPv4: `185.237.186.95/24`.
- IPv6: глобальный IPv6 не настроен.
- Шлюз: `185.237.186.1`.
- Внешний интерфейс: `eth0`.
- Часовой пояс: UTC; синхронизация времени активна.

## Сетевое состояние

- На внешних интерфейсах слушает только SSH на TCP 22.
- TCP 443 свободен.
- UDP 443 свободен.
- UFW неактивен.
- nftables не содержит правил.
- Политики iptables INPUT, FORWARD и OUTPUT установлены в ACCEPT.
- Локальная ОС не блокирует исходящие или входящие соединения.
- Возможный внешний firewall в панели провайдера этой проверкой не исключён.

## Программное окружение

Уже доступны:

- curl;
- wget;
- git;
- OpenSSL;
- Python 3.12.

Пока отсутствуют:

- gcc;
- make;
- Docker;
- Podman.

Это не является проблемой до окончательной проверки сети.

## Критический результат

DNS успешно разрешил `core.telegram.org` в IPv4 `149.154.167.99` и IPv6 `2001:67c:4e8:f004::9`.

При этом все три исходящих HTTPS-проверки к Telegram завершились тайм-аутом TCP-соединения через 7 секунд:

- `https://core.telegram.org/`;
- `https://core.telegram.org/getProxySecret`;
- `https://core.telegram.org/getProxyConfig`.

HTTP-код не был получен, TLS-рукопожатие не началось, удалённый IP в результате curl отсутствовал. Это означает, что соединение не дошло даже до TLS.

Поскольку запрос к `api.ipify.org` успешно вернул публичный IPv4, общий исходящий интернет работает. Предварительная гипотеза: маршрут или фильтрация российского провайдера/аплинка выборочно блокирует Telegram. Требуется расширенная проверка нескольких Telegram DC, контрольных HTTPS-ресурсов и маршрута.

## Информация о сети

Префикс `185.237.186.0/24` анонсируется из российского AS48347 (`JSC Mediasoft ekspert`). Это усиливает риск серверной фильтрации Telegram на российском участке сети.

## Домен

`prx.mailrhby.ru` на момент проверки с VPS не разрешается. Это ожидаемо, если старая CDN-запись уже удалена, а новая A-запись ещё не создана. До подтверждения пригодности VPS домен к нему не привязывать.

## Статус решения

**VPS пока не принят для установки Telegram-прокси.**

Положительные факторы:

- ресурсов более чем достаточно для одного пользователя;
- есть root-доступ;
- есть выделенный публичный IPv4;
- TCP/UDP 443 локально свободны;
- локальный firewall не мешает.

Блокирующий риск:

- исходящее TCP-соединение к `core.telegram.org:443` не устанавливается.

Следующий этап: безопасная матричная проверка общей сети, нескольких официальных Telegram адресов и маршрутов без установки программ и без изменения сервера.
