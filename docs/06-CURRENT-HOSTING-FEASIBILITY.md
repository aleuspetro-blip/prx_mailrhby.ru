# Возможность запуска Telegram-прокси на текущем сервере

Дата проверки: 20.07.2026

## Заключение

На текущем сервере виртуального хостинга Timeweb запустить рабочий Telegram MTProto Proxy или SOCKS5 Proxy без CDN нельзя.

## Причины

1. У пользователя нет root-доступа и sudo.
2. Файрвол виртуального хостинга отклоняет все входящие соединения, кроме портов стандартных служб.
3. Открыть отдельный TCP-порт для пользовательского процесса невозможно.
4. Порт 443 уже занят системным Nginx провайдера и не может быть передан пользовательскому MTProto-процессу.
5. Долговременные пользовательские службы на виртуальном хостинге отключены.
6. MTProto Proxy требует постоянно работающего TCP-процесса, доступного пользователям по внешнему адресу и порту.
7. SOCKS5 Proxy имеет те же базовые требования: внешний входящий TCP-порт и постоянно работающий процесс.

## Влияние отключения CDN

Удаление CDN и установка прямой A-записи на IP текущего хостинга не решит проблему. DNS только направит клиентов на общий IP сервера, но файрвол продолжит блокировать пользовательские TCP-порты, а порт 443 останется под управлением системного веб-сервера.

## Требуемая инфраструктура

Для проекта необходим отдельный VDS/VPS с:

- публичным IPv4;
- root-доступом;
- разрешёнными входящими TCP-соединениями;
- возможностью открыть порт 443 или другой порт;
- возможностью запускать постоянную службу;
- отсутствием запрета провайдера на прокси-сервисы.
