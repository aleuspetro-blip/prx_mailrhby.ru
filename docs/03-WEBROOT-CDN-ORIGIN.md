
В каталоге находится стандартная статическая страница:

```text

```

Параметры файла:

## CDN

Запрос к публичному домену возвращает:

```text
HTTP/2 200
x-cdn-edge-cache: HIT
content-length: 38873
```

DNS-цепочка:

```text
prx.mailrhby.ru
→ gykxozm5jw.cdn.twcstorage.ru
→ gykxozm5jw.a.trbcdn.net
→ 91.238.111.225
```

По официальной документации Timeweb Cloud CDN поддерживает только HTTP-методы `GET`, `HEAD` и `OPTIONS`. Подключить дополнительные методы невозможно. Следовательно, текущий CDN-домен не подходит как универсальная точка входа для проксирования `POST`, `PUT`, `PATCH`, `DELETE`, WebSocket или произвольного TCP-трафика.

Официальная документация:

- https://timeweb.cloud/docs/cdn/settings/http-headers
- https://timeweb.cloud/docs/cdn/settings/origin-and-domains
- https://timeweb.cloud/docs/cdn/settings/cache

## Проверка прямого origin

Запросы к `127.0.0.1` на порты 80 и 443 отклонены: локального слушателя в пользовательском пространстве нет.

Запросы к публичному адресу `92.53.96.141` с Host/SNI `prx.mailrhby.ru` возвращают парковочную страницу:

```text
HTTP/1.1 302 Moved Temporarily
Location: https://vh456.timeweb.ru/parking/?ref=prx.mailrhby.ru
```

Это означает, что `92.53.96.141` не подтвержден как прямой origin данного CDN-ресурса. Источник CDN может быть настроен на другой домен, IP или внутренний endpoint провайдера.
