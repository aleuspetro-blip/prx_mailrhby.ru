# CDN и обход блокировок Telegram

Дата: 20.07.2026

## Вывод

Обычный HTTP-CDN может помогать с доступностью веб-контента, но не может передавать нативный трафик Telegram MTProto, если не поддерживает проксирование произвольного TCP-трафика на уровне L4.

Текущий CDN Timeweb для `prx.mailrhby.ru` является HTTP-CDN. Согласно официальной документации Timeweb Cloud, через него поддерживаются только методы `GET`, `HEAD` и `OPTIONS`; подключить другие HTTP-методы нельзя. Документация описывает получение и кэширование HTTP-контента от источника, но не TCP-проксирование.

Официальный Telegram MTProxy запускается как отдельный процесс, принимает клиентские подключения на отдельном TCP-порту (в примере Telegram — порт 443) и требует прямого сетевого доступа к этому процессу.

Следовательно, текущая схема:

`Telegram client -> prx.mailrhby.ru -> Timeweb HTTP CDN -> virtual hosting`

не может работать как MTProto Proxy.

## Когда CDN теоретически может помочь

Нужен специализированный L4/TCP proxy или CDN, который одновременно:

- принимает произвольные TCP-соединения на порту 443;
- пересылает их без преобразования на origin-сервер;
- разрешает прокси-трафик Telegram правилами сервиса;
- предоставляет стабильный адрес или домен для подключения клиентов.

Обычного кэширования HTTP/HTTPS для этого недостаточно.

## Что текущий CDN можно использовать

- для страницы статуса;
- для инструкции по подключению;
- для публикации ссылки вида `tg://proxy?...`;
- для загрузки статических файлов проекта;
- для скрытия origin веб-сайта, но не origin MTProto Proxy.

## Рекомендуемая архитектура

`Telegram client -> prx.mailrhby.ru (A/AAAA без HTTP CDN) -> отдельный VPS -> MTProxy TCP/443`

Возможный дополнительный вариант — специализированный L4/TCP-сервис перед VPS, если его правила и технические возможности прямо разрешают такой трафик.

## Основные источники

- Официальный репозиторий TelegramMessenger/MTProxy на GitHub.
- Официальная документация Timeweb Cloud CDN: поддерживаемые HTTP-методы, назначение CDN, источник и домены раздачи.
