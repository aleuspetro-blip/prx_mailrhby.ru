# Назначение проекта: прокси для Telegram

Дата фиксации: 20.07.2026

## Цель

Создать прокси-сервер для подключения клиентских приложений Telegram.

Рабочее предположение: требуется MTProto Proxy, то есть прокси, который добавляется в Telegram по адресу сервера, порту и секрету. Telegram также поддерживает SOCKS5, но основной целевой вариант проекта — MTProto Proxy, пока владелец не укажет иное.

## Официальный источник Telegram

Основной технический источник: официальный репозиторий Telegram:

- https://github.com/TelegramMessenger/MTProxy

Официальный MTProxy требует:

- сборки исполняемого файла из исходного кода либо запуска официального контейнера;
- постоянно работающего процесса;
- входящего TCP-порта, обычно 443;
- собственного секрета для пользователей;
- загрузки `proxy-secret` и актуальной конфигурации Telegram;
- возможности управлять процессом и его автозапуском.

Пример официального запуска использует входящий порт 443 и отдельный локальный порт статистики 8888.

## Текущее окружение

Текущий сервер `vh456.timeweb.ru` является сервером виртуального хостинга. Учётная запись: `cj326260` без root-доступа.

Подтверждённые ограничения:

- root и sudo недоступны;
- Docker недоступен;
- системные службы создавать и изменять нельзя;
- входящие соединения разрешены только для стандартных служб провайдера;
- открыть произвольный входящий TCP-порт для пользовательского процесса нельзя;
- долговременные пользовательские службы отключены;
- правила Timeweb запрещают запуск прокси-серверов на виртуальном хостинге;
- CDN поддерживает только HTTP-методы GET, HEAD и OPTIONS и не является транспортом для MTProto/TCP.

## Вывод

Размещать MTProto Proxy или SOCKS5-прокси на текущем виртуальном хостинге нельзя технически и по правилам услуги. Попытки обходить ограничения не предпринимаются.

Для проекта потребуется отдельный VDS/VPS с:

- root-доступом;
- отдельным публичным IPv4-адресом;
- разрешённым входящим TCP-портом;
- возможностью запустить постоянную службу;
- возможностью направить `prx.mailrhby.ru` напрямую на IP сервера без HTTP CDN.

Текущий виртуальный хостинг и рабочие Python-процессы не изменяются. Каталог `/home/c/cj326260/public_html` не используется для нового прокси-проекта. Текущий CDN и каталог `/home/c/cj326260/mailrhby.ru/public_html` могут позднее использоваться только для информационной веб-страницы прокси, но не для передачи Telegram-трафика.
