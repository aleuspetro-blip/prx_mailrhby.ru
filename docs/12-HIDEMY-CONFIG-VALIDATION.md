## Риск

Исходная конфигурация является полным IPv4-туннелем. Её нельзя запускать напрямую на удалённом VPS, поскольку автоматическое изменение default route может нарушить обратный маршрут SSH и входящих подключений к будущему MTProto Proxy.

## Безопасный план

1. Создать отдельную рабочую копию конфигурации.
2. Удалить директиву DNS.
3. Добавить `Table = off`, чтобы `awg-quick` не создавал маршруты автоматически.
4. Поднять интерфейс только для проверки handshake.
5. Временно добавить один host route к `core.telegram.org` через туннель.
6. Проверить доступ к Telegram.
7. Удалить временный маршрут и выключить интерфейс.
8. После успешного теста настроить policy routing только для отдельного пользователя процесса Telegram-прокси.
